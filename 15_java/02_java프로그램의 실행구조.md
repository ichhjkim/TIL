# 02_java프로그램의 실행구조



## 2-1: 환경변수 설정

bin > `java.exe `: JVM 구동 명령

bin> `javac.exe` : 컴파일러



## 2-2: Java 컴파일러와 JVM



java소스 작성 부터 프로그램 실행까지의 순서



java소스(xxx.java) --> 바이트 코드 파일(xxx.class)  -> 기계어 -> 실행	   

​		java컴파일러(javac.exe)   JVM구동(java.exe) --> LINK되었따라고 함. (link되면 메모리로딩/실행준비/실행결정/초기화)



## 2-3: 가비지 컬렉터(gabage collector)



프로그램 실행에 필요한 메모리를 가비지 컬렉터가 자동으로 관리



| c계열 프로그램                                               | Java프로그램                                                 |
| ------------------------------------------------------------ | ------------------------------------------------------------ |
| 개발자가 직접메로리를 관리해야함. 만약 메모리 관리를 잘못할 경우 메모리 누수가 발생하고 타프로그램 동작이 멈출 수 있음. | 개발자가 메모리에 접근할 수 없음. 따라서 개발자는 메모리 관리를 할 수 없고 가비지 컬렉터가 불필요한 메모리를 회수해서 메모리를 최적화함 |
