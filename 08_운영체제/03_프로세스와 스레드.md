# 03_프로세스와 스레드



## 01_프로세스의 개념과 상태변화



### 1. 프로세스의 개념

> 다중 프로그램 환경에서는 여러 프로그램을 메모리에 적재하여 병행 실행할 수 있어 컴퓨터의 효율을 높일 수 있다. 



#### 프로세스 용어

- 실행중인 프로그램
- 비동기적 행위
- 실행 중인 프로시저
- 실행중인 프로시저의 제어 추적
- 운영체제에 들어있는 프로세스 제어 블록
- 프로세서에 할당하여 실행할 수 있는 개체 디스패치가 가능한 대상



일반적인 프로세스 정의는 실행중인 프로그램

디스크에 있던 프로그램을 메모리에 적재해 *운영체제의 제어*를 받는 상태가 된것. 자신만의 `메모리 영역`(주소 공간)이 있음을 의미.  프로세스는 `프로그램 실행의 인스턴스`도 된다.



>  웹 브라우저 창을 한개 열면 프로세스 한개, 웹브라우저 창을 2개 열면 프로세스 2개. 



프로세스가 실행중인 프로그램이 되려면

- 프로세서 점유시간
- 메모리
- 파일
- 입출력 장치

등 자원이 필요. 자원은 프로세스를 실행하거나 생성할 떄 할당된다. 



 

| 프로그램                                                     | 프로세스                                                     |
| ------------------------------------------------------------ | ------------------------------------------------------------ |
| 컴파일한 코드, 초기화 전역변수 처럼 정적 데이터 포함. 정적인 개체 | 메모리 구조, 프로그램 카운터나 레지스터처럼 현재 어떤 자원을 사용하는지 관련 정보가 들어있는 동적인 개체 |
