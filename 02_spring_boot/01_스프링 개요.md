# 01_스프링 애플리케이션



- 웹 아카이브(web archive(WAR))

  - WEB-INF 폴더 필요> 하위에 lib, classes, 
    - lib > third party library
    - classes > web application class
  - 필요시, JSP/HTML/CSS

  - 패키징 by 유틸리티 - 예) 메이븐, 그레이들(==그루비DSL라고들 부름)
  - 서버 : 톰캣, 제티, 제이보스, 웹스피어, J2EE



#### 스프링부트는 @RestController @RequestMapping 같은 애너테이션이 나오면 웹 애플리케이션으로 간주. 톰캣서버는 품안에 지니고 있어 자체 실행가능



## 01_왜 스프링 부트인가?

- 12-팩터앱(12Factor App)가이드를 준수한 클라우드 네이티브 애플리케이션(넷플릭스 기술팀) 

- 개발과 배포 시간을 줄여 생산성을 높임
- 바로 제품 출시가 가능한 엔터프라이즈 스프링 앱 개발
- 스프링 부트 Actuator(상태 체크 등 앱관리를 도와주는 모듈) 및 웹 애플리케이션을 실행할 내장 컨테이너(톰캣, 언더토우, 제티 등) 같은 비기능적 요건 충족

## 03_CLI 명령어

실행

`spring run [옵션] <파일> [--] [인자]`



### 그루비 애플리케이션

```bash
$ spring run app.groovy <-- 기본 8080포트 리스닝
$ spring run app.groovy --server.port=8888
$ spring run -cp lib/mylib.jar app.groovy <-- 서드파티 라이브러리 추가
$ spring run WebApp.java  --> 자바 애플리케이션 코드 실행하는 명령
$ spring run *.groovy  <-- 와일드카드 문자(*)로 한번에 컴파일
$ JAVA_OPTS=-Xmx2g spring run app.groovy
```



#### test 명령어

`spring test [옵션] 파일 [---] [인자]`



#### grab/jar 명령어

스프링 그루피 스크립트 및 자바 의존체를 가져와 

`spring grab/jar [옵션] 파일 [--] 인자`



#### install/uninstall 명령어

spring install/uninstall [옵션] <식별자>



#### init명령어

스프링 이니셜라이저로 새 프로젝트를 만든다.

`spring init [옵션] [경로]`

spring inir -d=web,jdbc --build=gradle --> 그레이들 프로젝트 생성

`spring init --list` : 기타 의존체나 파라미터 값은 --list로 조회
